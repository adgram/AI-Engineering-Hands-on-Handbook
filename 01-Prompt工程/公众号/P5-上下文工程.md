# Prompt工程 | P5：上下文工程

## 准备

P2提到过，模型之所以能“记住”之前说过什么，是因为我们把完整的对话历史和当前问题打包成了一个整体发送过去，这个整体就是模型的“**上下文**”。而在长时间的多轮对话中，上下文太长不仅会超出模型输入上限，也会因大量输入与当前对话无关，给模型造成干扰。本节的目的便是如何收集、筛选、排列和压缩上下文。

## GSSC 流水线

GSSC 是上下文工程的核心工作流：Gather → Select → Structure → Compress。

### G（收集）

收集提示词、历史对话、查询的结果等相关信息：

| 来源 | 类型 | 优先级 |
|------|------|--------|
| System Prompt | 固定指令 | 最高（始终保留） |
| 记忆系统 | 用户画像/长期偏好 | 高 |
| RAG 检索结果 | 外部知识 | 中 |
| 对话历史 | 本轮交互记录 | 低（可压缩） |

### S（筛选）

从收集到的信息中挑出与当前问题最相关的部分，在 Token 预算有限时优先保留下有价值的，丢掉无关或低价值的。

1. 算分 — 每条信息与当前 query 的相关性
2. 截断 — 低于阈值的扔掉；超出 budget 时逐步提高阈值，直到数量达标

下面示例给每个信息片段计算与当前消息的 Relevance Score（0-1），预算紧张时提高阈值。

```python
class RelevanceScorer:
    def __init__(self, threshold=0.3):
        self.threshold = threshold

    def score(self, text: str, query: str) -> float:
        text_words = set(text.lower().split())
        query_words = set(query.lower().split())
        if not query_words:
            return 0.5
        return len(text_words & query_words) / len(query_words)

    def filter(self, items: List[Dict], query: str, budget: int = None) -> List[Dict]:
        for item in items:
            item["relevance"] = self.score(item["content"], query)
        scored = sorted(items, key=lambda x: x["relevance"], reverse=True)
        result = [x for x in scored if x["relevance"] >= self.threshold]
        if budget:
            while len(result) > budget and self.threshold < 1.0:
                self.threshold += 0.1
                result = [x for x in scored if x["relevance"] >= self.threshold]
        return result

scorer = RelevanceScorer()
items = [
    {"id": 1, "content": "Python 是一种编程语言"},
    {"id": 2, "content": "今天天气很好"},
    {"id": 3, "content": "Python 常用于数据科学和 Web 开发"},
]
for item in scorer.filter(items, "Python 开发"):
    print(f"  [{item['relevance']:.2f}] {item['content']}")
```

**`score()`** — 基于关键词匹配：

把 query 和文本都按空格切词，算**交集词数 / query 词数**。

注意：中文靠 `.split()` 切词效果很差（整句当一个词），这只是教学演示用的简化版。生产环境会用 embedding 语义相似度替代。

**`filter(budget=N)`** — 阈值逐步收紧：

1. 先按 relevance 排序，筛掉低于 threshold (默认0.3) 的
2. 如果结果数还超过 budget → 每轮 +0.1 阈值 → 直到数量达标或阈值为1.0

### S（结构化）

**Lost in the Middle 现象**：在长上下文中，模型倾向于记住开头和结尾的信息，中间部分容易被忽略。

利用注意力 U 型曲线，关键定义放开头，最新问题放结尾，参考信息放中间。

```python
def structure_context(system_prompt, definitions, references, user_input):
    messages = [{"role": "system", "content": system_prompt}]
    if definitions:
        messages.append({"role": "system", "content": f"## 关键定义\n{definitions}"})
    if references:
        messages.append({"role": "system", "content": f"## 参考资料\n{references}"})
    messages.append({"role": "user", "content": user_input})
    return messages
```

### C（压缩）

在筛选之后，把保留下来的内容进一步缩小体积。

| 方式                   | 性质 | 做法                                                         |
| :--------------------- | :--- | :----------------------------------------------------------- |
| **紧凑化（优先）**     | 可逆 | 把长内容存到文件系统，只留路径引用（Manus / Claude Code 的做法） |
| **结构化压缩**         | 无损 | 丢掉冗余字段，如 `debug_info`、`raw_response`                |
| **摘要化（最后手段）** | 有损 | 让 LLM 把内容浓缩成摘要，原始内容备份到文件以便恢复          |

```
优先顺序：紧凑化（可逆） → 结构化压缩（无损） → 摘要化（有损，最后手段）
```

```python
class Compressor:
    def compress_summary(self, text: str) -> str:
        resp = client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=[{"role": "user", "content": f"压缩到 50 字以内：\n{text}"}]
        )
        return resp.choices[0].message.content

    def compress_structured(self, data: Dict) -> str:
        essential = {k: v for k, v in data.items() if k not in ["debug_info", "raw_response"]}
        return json.dumps(essential, ensure_ascii=False, sort_keys=True)

    def selective_compress(self, items: List[Dict], threshold=0.7) -> List[str]:
        return [
            item["content"] if item.get("relevance", 0) >= threshold
            else f"[压缩] {self.compress_summary(item['content'])}"
            for item in items
        ]
```

## 三种经典策略（GSSC 视角）

### 策略 1：滑动窗口 — GSSC 的"只 Select + Structure"

**原理**：只保留最近 N 轮对话，丢弃中间历史。实现简单，但同时也丢失了所有早期信息。

**适用场景**：客服对话、单轮问答、多轮但历史无关的场景。每次提问仅依赖当前上下文，不需要回顾早期的信息。

**优点**：实现极简、无额外 API 开销、内存占用低。**缺点**：丢弃所有早期信息，无法利用历史积累的上下文；对话轮数超出窗口后，模型不再"认识"用户之前的偏好或说过的话。

```python
class SlidingWindow:
    def __init__(self, max_rounds=5, system_prompt=None):
        self.max_rounds = max_rounds
        self.messages = []
        if system_prompt:
            self.messages.append({"role": "system", "content": system_prompt})

    def ask(self, user_input):
        self.messages.append({"role": "user", "content": user_input})
        while len(self.messages) > self.max_rounds * 2 + 1:
            if self.messages[1]["role"] == "user":
                del self.messages[1:3]
            else:
                break
        response = client.chat.completions.create(model="deepseek-v4-flash", messages=self.messages)
        reply = response.choices[0].message.content
        self.messages.append({"role": "assistant", "content": reply})
        return reply
```

> GSSC 视角：Gather 仅为对话历史；Select 用时间（最近 N 轮）；Structure 天然 U 型；Compress 不做。

### 策略 2：摘要压缩 — GSSC 的"Compress + Summarization"

**原理**：用 LLM 对长对话历史做摘要，保留语义要点但大幅压缩 token。代码中通过 Token 预算触发：实际上下文超过 `max_tokens` 时，将最近历史用 LLM 压缩为摘要放入 system prompt，清空近期记录。

**适用场景**：长时间跨度的对话（几小时到几天）、Agent 连续执行多个任务的日志、需要回顾早期内容但 token 预算有限的场景。

**优点**：能把无限长的对话历史压进固定 token 预算内。**缺点**：摘要是有损压缩，原始细节不可恢复；压缩本身消耗一次 API 调用；摘要质量依赖 LLM 能力，可能遗漏关键信息。

```python
class SummarizationStrategy:
    def __init__(self, max_tokens=2000):
        self.max_tokens = max_tokens
        self.summary = ""
        self.recent = []

    def ask(self, user_input):
        self.recent.append({"role": "user", "content": user_input})
        context = [{"role": "system", "content": "你是一个助手"}]
        if self.summary:
            context.append({"role": "system", "content": f"历史摘要：{self.summary}"})
        context.extend(self.recent)

        total = int(sum(len(m["content"]) for m in context) * 1.5)
        if total > self.max_tokens:
            history = "\n".join(f"{m['role']}: {m['content']}" for m in self.recent)
            resp = client.chat.completions.create(
                model="deepseek-v4-flash",
                messages=[{"role": "user", "content": f"压缩对话：\n{history}"}]
            )
            self.summary = resp.choices[0].message.content
            self.recent = []
            context = [{"role": "system", "content": f"历史摘要：{self.summary}"},
                       {"role": "user", "content": user_input}]

        response = client.chat.completions.create(model="deepseek-v4-flash", messages=context)
        reply = response.choices[0].message.content
        self.recent.append({"role": "assistant", "content": reply})
        return reply
```

> GSSC 视角：Gather 为 System + 摘要 + 最近对话；Select 用 Token 预算做阈值；Compress 用摘要化。

### 策略 3：向量检索 — GSSC 的"Gather + Semantic Select"

**原理**：将对话历史向量化存储，每次用户提问时用 query 的 embedding 从历史中检索语义最相关的若干条，放入上下文。代码中用 MD5 哈希模拟 embedding 生成（生产中应替换为真实 embedding 模型如 text-embedding-3）。

**适用场景**：知识密集型对话、用户反复跨话题切换、需要从大量历史中精准定位之前说过内容的场景。

**优点**：Select 精度最高，能从长篇历史中只挑出真正相关的内容，不浪费 token 在无关历史上。**缺点**：需要 embedding 服务（额外 API 调用和成本）；检索质量依赖 embedding 模型质量；哈希模拟不可用于生产，仅用于演示 GSSC 流程。

```python
class VectorRetrievalStrategy:
    def __init__(self, system_prompt=None):
        self.system_prompt = system_prompt
        self.history = []

    def _get_embedding(self, text: str) -> list:
        import hashlib
        return [float(hashlib.md5(text.encode()).hexdigest(), 16) % 10 / 10]

    def retrieve(self, query: str, top_k=3) -> list:
        query_emb = self._get_embedding(query)
        scored = []
        for h in self.history:
            sim = sum(abs(a - b) for a, b in zip(query_emb, self._get_embedding(h["content"])))
            scored.append((sim, h["content"]))
        scored.sort(reverse=True)
        return [c for _, c in scored[:top_k]]

    def ask(self, user_input):
        relevant = self.retrieve(user_input)
        messages = []
        if self.system_prompt:
            messages.append({"role": "system", "content": self.system_prompt})
        if relevant:
            messages.append({"role": "system", "content": "相关历史：\n" + "\n".join(relevant)})
        messages.append({"role": "user", "content": user_input})
        response = client.chat.completions.create(model="deepseek-v4-flash", messages=messages)
        self.history.append({"role": "user", "content": user_input})
        self.history.append({"role": "assistant", "content": response.choices[0].message.content})
        return response.choices[0].message.content
```

> GSSC 视角：Gather 为对话历史；Select 用语义相似度（最精准）；Structure 把相关历史放中间靠前。

### 三种策略对比

| 策略 | Gather | Select | Structure | Compress |
|------|--------|--------|-----------|----------|
| 滑动窗口 | 对话历史 | 时间（最近 N 轮） | U 型曲线 | 无 |
| 摘要压缩 | 对话+摘要 | Token 预算 | 前摘要+后当前 | 摘要化 |
| 向量检索 | 对话历史 | 语义相似度 | 相关历史居中 | 无 |
