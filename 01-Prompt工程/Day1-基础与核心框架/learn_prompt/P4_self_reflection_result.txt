=== 第一版（初始实现） ===
下面是一个从 JSON 配置文件读取数据库连接信息和 SQL 查询，执行并返回结果的 Python 函数。它使用 `pymysql` 连接 MySQL 数据库，并以字典列表的形式返回查询结果。

```python
import json
import pymysql
from pymysql import cursors
from typing import List, Dict, Any

def execute_query_from_json(json_path: str) -> List[Dict[str, Any]]:
    """
    从 JSON 文件中读取数据库配置和 SQL 查询，连接数据库执行查询并返回结果。

    JSON 文件应包含以下字段：
        host (str): 数据库主机
        port (int): 端口
        user (str): 用户名
        password (str): 密码
        database (str): 数据库名
        query (str): 要执行的 SQL 语句
        charset (str, optional): 字符集，默认 utf8

    参数:
        json_path (str): JSON 配置文件的路径

    返回:
        List[Dict[str, Any]]: 查询结果，每行是一个字典，字段名为列名

    异常:
        FileNotFoundError: 配置文件不存在
        ValueError: JSON 格式错误
        KeyError: 缺少必要配置项
        RuntimeError: 数据库连接或 SQL 执行出错
    """
    # 1. 读取 JSON 配置
    try:
        with open(json_path, 'r', encoding='utf-8') as f:
            config = json.load(f)
    except FileNotFoundError:
        raise FileNotFoundError(f"配置文件 {json_path} 不存在")
    except json.JSONDecodeError as e:
        raise ValueError(f"JSON 解析错误: {e}")

    # 2. 检查必要字段
    required_keys = ['host', 'port', 'user', 'password', 'database', 'query']
    missing = [key for key in required_keys if key not in config]
    if missing:
        raise KeyError(f"配置缺少必要字段: {missing}")

    # 3. 连接数据库并执行查询
    connection = None
    cursor = None
    try:
        connection = pymysql.connect(
            host=config['host'],
            port=int(config['port']),
            user=config['user'],
            password=config['password'],
            database=config['database'],
            charset=config.get('charset', 'utf8'),
            cursorclass=pymysql.cursors.DictCursor  # 返回字典
        )
        with connection.cursor() as cursor:
            cursor.execute(config['query'])
            results = cursor.fetchall()
        return results
    except pymysql.MySQLError as e:
        raise RuntimeError(f"数据库操作失败: {e}")
    finally:
        # 确保连接关闭
        if connection:
            connection.close()
```

## 使用示例

假设有一个 `config.json` 文件：

```json
{
    "host": "localhost",
    "port": 3306,
    "user": "root",
    "password": "123456",
    "database": "testdb",
    "query": "SELECT id, name, age FROM users WHERE age > 18"
}
```

调用函数：

```python
result = execute_query_from_json("config.json")
for row in result:
    print(row)
# 输出类似：[{'id': 1, 'name': '张三', 'age': 25}, {'id': 2, 'name': '李四', 'age': 30}]
```

## 注意事项

- 需要安装 `pymysql`：`pip install pymysql`
- 如果连接其他数据库（如 PostgreSQL），可将数据库驱动替换为 `psycopg2` 或 `sqlalchemy`，但本例专注于 MySQL。
- 建议在正式使用时对 SQL 语句进行参数化防止注入，但此函数直接执行 `config['query']`，适用于内部可信配置。如需支持参数，可以扩展 JSON 配置增加 `params` 字段并在 `execute` 时传入。

## 问题列表

### 1. SQL 注入风险（严重程度：**高**）

**原因**：  
函数直接执行 `cursor.execute(config['query'])`，将 JSON 文件中的 SQL 语句视为可信输入。若攻击者能够修改该 JSON 文件（例如通过网络、文件权限不足等），则可以注入任意 SQL 命令，导致数据泄露、篡改或删除。虽然注释提到“适用于内部可信配置”，但作为通用函数，应默认采用参数化查询来防御注入。

**修复方案**：  
- 在 JSON 配置中增加可选的 `params` 字段（列表或字典），并在 `execute` 时传入参数。  
- 对于无需参数的情况，仍可允许直接执行，但建议对 SQL 进行白名单校验或至少告知用户风险。  
- 更安全的做法是强制使用参数化查询，但考虑到灵活性，可以在文档中明确安全要求。

---

### 2. `port` 类型转换可能引发 `ValueError`（严重程度：**中**）

**原因**：  
代码中使用 `port=int(config['port'])` 将端口转换为整数。若 JSON 中的 `port` 值不是整数（例如字符串 `"abc"` 或浮点数 `3306.5`），`int()` 会抛出 `ValueError`，且该异常没有被捕获，导致函数直接崩溃，且用户看到的异常信息不友好。

**修复方案**：  
- 直接使用 `config['port']`（JSON 解析后原本就是整数，不需要转换）。  
- 如果仍要转换，应捕获 `ValueError` 并抛出明确的 `ValueError` 或 `ConfigError`。

---

### 3. 异常文档与实际抛出不一致（严重程度：**低**）

**原因**：  
文档字符串声明会抛出 `ValueError`（JSON 格式错误）和 `KeyError`（缺少配置项），但若 `port` 转换失败也会抛出 `ValueError`，而文档未说明此情形。另外，文档未提及 `pymysql.MySQLError` 被包装为 `RuntimeError`，但已有说明，故此处轻微不一致。

**修复方案**：  
- 统一异常体系：自定义配置错误异常，或细化文档。

---

### 4. 未验证 `query` 字段类型（严重程度：**低**）

**原因**：  
假设 `query` 是字符串，但若 JSON 中该字段为数字、列表等，`cursor.execute` 会抛出类型错误，错误信息不够清晰。添加类型检查可提高健壮性。

**修复方案**：  
- 在验证必要字段时，增加对 `query` 类型的检查，例如 `isinstance(config['query'], str)`。

---

### 5. 冗余导入（严重程度：**低**）

**原因**：  
`from pymysql import cursors` 被导入但未使用（程序中仅使用了 `pymysql.cursors.DictCursor`，但已通过 `pymysql.cursors` 访问）。虽然不影响运行，但属于代码清理问题。

**修复方案**：  
- 移除未使用的导入。

---

### 6. 资源管理：潜在未关闭的 cursor（严重程度：**低**）

**原因**：  
代码使用 `with connection.cursor() as cursor` 来确保 cursor 被正确关闭，这是正确的。但若在创建 cursor 之前（例如 `connection` 刚建立）就发生异常，`finally` 块中 `connection.close()` 仍会执行，没有问题。实际上资源管理良好，但有一个次要细节：`with` 块中的 `cursor` 变量被 `finally` 中的 `cursor` 覆盖（`cursor = None` 在前），但这不影响行为。完全没问题。

---

### 7. 缺少连接超时设置（严重程度：**低**）

**原因**：  
未指定 `connect_timeout` 或 `read_timeout`，在网络异常或数据库不可用时，连接可能长时间阻塞。虽然不是功能缺陷，但生产环境中建议设置合理的超时。

**修复方案**：  
- 在 `pymysql.connect()` 中添加 `connect_timeout=5` 等参数。

## 修复后的完整版本

```python
import json
from typing import List, Dict, Any, Optional

import pymysql
from pymysql.cursors import DictCursor


def execute_query_from_json(
    json_path: str,
    *,
    default_charset: str = "utf8",
    connect_timeout: int = 5,
    read_timeout: int = 10,
) -> List[Dict[str, Any]]:
    """
    从 JSON 文件中读取数据库配置和 SQL 查询，连接数据库执行查询并返回结果。

    JSON 文件应包含以下字段：
        host (str): 数据库主机
        port (int): 端口（整数）
        user (str): 用户名
        password (str): 密码
        database (str): 数据库名
        query (str): 要执行的 SQL 语句
        params (list or dict, optional): 参数化查询的参数，默认 None
        charset (str, optional): 字符集，默认 utf8

    参数:
        json_path (str): JSON 配置文件的路径
        default_charset (str): 默认字符集，用于配置文件中未指定时的回退
        connect_timeout (int): 连接超时秒数
        read_timeout (int): 读取超时秒数

    返回:
        List[Dict[str, Any]]: 查询结果，每行是一个字典，字段名为列名

    异常:
        FileNotFoundError: 配置文件不存在
        ValueError: JSON 格式错误 或 配置值类型错误
        KeyError: 缺少必要配置项
        RuntimeError: 数据库连接或 SQL 执行出错
    """
    # 1. 读取 JSON 配置
    try:
        with open(json_path, 'r', encoding='utf-8') as f:
            config = json.load(f)
    except FileNotFoundError:
        raise FileNotFoundError(f"配置文件 {json_path} 不存在")
    except json.JSONDecodeError as e:
        raise ValueError(f"JSON 解析错误: {e}") from e

    # 2. 检查必要字段及其类型
    required_keys = ['host', 'port', 'user', 'password', 'database', 'query']
    missing = [key for key in required_keys if key not in config]
    if missing:
        raise KeyError(f"配置缺少必要字段: {missing}")

    # 对类型进行简单校验
    if not isinstance(config['query'], str):
        raise ValueError("'query' 字段必须为字符串")
    if not isinstance(config['port'], int):
        # JSON 中的数字解析后即为 int，但防御性检查
        raise ValueError("'port' 字段必须为整数")
    # 可选的 params 字段类型校验
    if 'params' in config and not isinstance(config['params'], (list, dict)):
        raise ValueError("'params' 字段必须为列表或字典")

    # 3. 连接数据库并执行查询
    connection = None
    try:
        connection = pymysql.connect(
            host=config['host'],
            port=config['port'],  # 直接使用，无需 int() 转换
            user=config['user'],
            password=config['password'],
            database=config['database'],
            charset=config.get('charset', default_charset),
            cursorclass=DictCursor,
            connect_timeout=connect_timeout,
            read_timeout=read_timeout,
        )

        with connection.cursor() as cursor:
            # 参数化查询：如果提供了 params 则传递，否则为 None
            params = config.get('params')
            if params is None:
                cursor.execute(config['query'])
            else:
                cursor.execute(config['query'], params)
            results = cursor.fetchall()
        return results

    except pymysql.MySQLError as e:
        raise RuntimeError(f"数据库操作失败: {e}") from e
    except Exception as e:
        # 捕获其他意外异常（如端口转换、类型错误等），统一包装为 RuntimeError
        # 注意：此处的异常链会保留原始异常信息
        raise RuntimeError(f"执行查询时发生未预期错误: {e}") from e
    finally:
        if connection:
            try:
                connection.close()
            except pymysql.MySQLError:
                # 关闭连接时的异常一般可忽略，但记录日志更好
                pass
```

### 主要改进说明

1. **安全性**：支持通过 JSON 中的 `params` 字段进行参数化查询，防御 SQL 注入。若不需要参数，`params` 可省略。
2. **类型安全**：移除不必要的 `int()` 转换，直接使用 JSON 解析结果；添加了类型校验并对非法类型抛出明确的 `ValueError`。
3. **超时设置**：增加了 `connect_timeout` 和 `read_timeout` 参数，避免无限阻塞。
4. **异常处理**：
   - 使用 `from e` 保留异常链，便于调试。
   - 在 `finally` 中关闭连接时捕获可能的异常，确保不会因关闭失败而覆盖主异常。
   - 将意外的异常（如类型错误）捕获并包装为 `RuntimeError`，保持接口一致性。
5. **代码清理**：移除了未使用的 `cursors` 导入。
6. **文档更新**：补充了 `params`、超时参数以及异常信息的说明。

此版本在保持原有功能的同时，显著提升了安全性、健壮性和可维护性。
