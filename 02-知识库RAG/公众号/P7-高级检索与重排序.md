# 知识库RAG | P7：高级检索与重排序

## 前言

前文中的单路检索存在一些局限：短查询语义不足、专业术语难命中、Top-K 排序不够精细。本节介绍两个提升检索质量的高级技术——**HyDE 与多路召回**提升召回率，**Rerank 重排序**提升排序精度。

这是"粗筛→精排"两阶段架构的体现：先用快速方法召回足够多的候选，再用精细模型重新排序选出最优。

## HyDE：假设文档嵌入

### 核心思想

短查询（如"RAG 是什么"）的向量语义信息稀薄，和长文档的向量不在一个"密度"上，导致检索效果差。**HyDE（Hypothetical Document Embeddings）** 的思路是"先假设再检索"——让 LLM 先生成一个 100-200 字的假设答案，再用这个假设答案（而非原查询）去向量库检索。

```
原查询："RAG 是什么"（向量稀疏）
    ↓ LLM 生成假设答案
假设答案："RAG 是检索增强生成，通过先检索知识库再生成回答..."（向量丰富）
    ↓ 用假设答案检索
命中更相关的文档
```

### 适用场景

| 查询类型 | HyDE 效果 |
|---------|----------|
| 短查询（几个词） | 很好 |
| 专业术语 | 好 |
| 表述不清 | 明显提升 |
| 长查询（已经很详细） | 提升有限 |

```python
class HyDERetriever:
    def __init__(self, store, llm, model="deepseek-v4-flash"):
        self.store = store
        self.llm = llm
        self.model = model

    def generate_hypothetical(self, query: str) -> str:
        """让 LLM 生成假设答案"""
        resp = self.llm.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": f"请用 100-200 字回答这个问题（用于检索，不需要完全准确）：\n{query}"}]
        )
        return resp.choices[0].message.content

    def search(self, query: str, k: int = 3) -> dict:
        hypo_doc = self.generate_hypothetical(query)
        # 用假设答案而非原查询检索
        return self.store.search(hypo_doc, n_results=k)
```

缺点：每次检索多一次 LLM 调用，延迟和成本增加。适合对召回率要求高的场景。

## 多路召回 + RRF 融合

单路检索总有盲区。多路召回同时执行向量搜索、关键词搜索、HyDE 搜索，再用 RRF 融合（P4 已介绍），取长补短：

```python
class MultiRouteRetriever:
    """三路检索 + RRF 融合"""
    def __init__(self, store, llm, model="deepseek-v4-flash"):
        self.store = store
        self.hyde = HyDERetriever(store, llm, model)
        # 关键词索引（简化版，生产用 BM25）
        self.keyword_index = {}

    def vector_search(self, query, k=10):
        return self.store.search(query, n_results=k)['ids'][0]

    def keyword_search(self, query, k=10):
        # 简单包含匹配，生产环境替换为 BM25
        results = [(doc_id, text) for doc_id, text in self.keyword_index.items() if query in text]
        return [r[0] for r in results[:k]]

    def multi_route_search(self, query: str, k: int = 5) -> dict:
        # 三路并行检索
        vector_results = self.vector_search(query, k * 2)
        keyword_results = self.keyword_search(query, k * 2)
        hyde_results = self.hyde.search(query, k * 2)['ids'][0] if hasattr(self.hyde.search(query, k), 'ids') else []

        # RRF 融合
        scores = {}
        for results in [vector_results, keyword_results, hyde_results]:
            for rank, doc_id in enumerate(results, 1):
                scores[doc_id] = scores.get(doc_id, 0) + 1 / (60 + rank)

        fused = sorted(scores.items(), key=lambda x: x[1], reverse=True)[:k]
        return {"fused_ranking": fused, "routes": {
            "vector": vector_results[:3],
            "keyword": keyword_results[:3],
            "hyde": hyde_results[:3]
        }}
```

## Rerank 重排序

向量搜索是"快速初筛"——用 ANN 算法从海量文档中快速捞出候选，但缺乏 pairwise（两两对比）的细致比较。可能排名并不一定遵循严格的相关性。

### 两种 Rerank 方法

**方法 1：LLM Rerank**

逐个文档让 LLM 打 0-10 分，精度高但慢、成本高，适合小规模或离线场景。

**方法 2：API Rerank（推荐）**

调用专门的排序模型进行，如 SiliconFlow 的 `BAAI/bge-reranker-v2-m3` 模型：

```python
class ApiReranker:
    """调用 Cross-Encoder API 重排序"""
    def __init__(self, api_key, base_url="https://api.siliconflow.cn/v1"):
        self.client = OpenAI(api_key=api_key, base_url=base_url)

    def rerank(self, query: str, documents: list, top_n: int = 5) -> list:
        """返回重排序后的 [(doc, score), ...]"""
        response = self.client.embeddings.create(
            model="BAAI/bge-reranker-v2-m3",
            input={"query": query, "documents": documents}  # Cross-Encoder 输入
        )
        scored = [(documents[i], s) for i, s in enumerate(response.data[0].embedding)]
        return sorted(scored, key=lambda x: x[1], reverse=True)[:top_n]
```

## 完整的检索优化链路

把本节技术串起来，就是从 Naive RAG 升级到 Advanced RAG 的路径：

```
Naive RAG（单路向量检索，Top-K 直接用）
    ↓ 召回优化
+ HyDE（假设文档，提升短查询召回）
+ 多路召回（向量+关键词+HyDE，RRF 融合）
    ↓ 精排优化
+ Rerank（Cross-Encoder 重排序，提升 Top-K 质量）
    ↓ 评估驱动
+ 质量评估（P6 的 HR/MRR/Faithfulness 量化对比）
```
