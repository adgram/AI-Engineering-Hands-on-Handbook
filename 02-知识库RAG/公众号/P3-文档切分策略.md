# 知识库RAG | P3：文档切分策略

## 前言

P1 解决了"文本如何变向量"，P2 解决了"向量如何存储和检索"。但在处理文本前还需要考虑如何**文档切分（Chunking）**。

一篇万字长文不能整篇作为一个向量（超出模型输入上限，且检索粒度太粗），也不能按单字切（语义全无）。切分粒度直接影响检索质量：切太大，检索到的片段包含大量无关内容；切太小，语义不完整，模型难以理解。本节介绍六种切分策略及其适用场景。

切分的核心目标：**让每个片段语义完整且聚焦单一主题**，这样检索时才能精准命中。

## 策略 1：固定长度切分

最简单的方式，按固定字符数切分并带重叠（overlap 防止切断语义）：

```python
def fixed_length_chunk(text: str, chunk_size: int = 200, overlap: int = 20) -> list:
    chunks = []
    start = 0
    while start < len(text):
        chunk = text[start:start + chunk_size]
        if chunk:
            chunks.append(chunk.strip())
        start += (chunk_size - overlap)
    return chunks
```

- **优点**：简单、可预测
- **缺点**：可能切断句子/段落，破坏语义完整性

## 策略 2：递归字符切分（推荐）

LangChain 提供的 `RecursiveCharacterTextSplitter`，按优先级分隔符逐级切分——先尝试按段落切，段落太大再按行切，行太大再按句切：

```bash
pip install langchain-text-splitters
```

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=100,
    chunk_overlap=20,
    separators=["\n\n", "\n", "。", "，", " ", ""]  # 由大到小
)

chunks = splitter.split_text(text)
```

这是最常用的策略，兼顾语义完整性和长度控制。

## 策略 3：基于规则的段落切分

按标题和段落边界切分，不跨语义单元，适合有清晰结构的文档（技术文档、规范标准）：

```python
import re

def rule_based_chunk(text: str) -> list:
    sections = re.split(r'(?=##\s)', text)  # 按 ## 标题切分
    chunks = []
    for section in sections:
        if not section.strip():
            continue
        paragraphs = re.split(r'\n\s*\n', section.strip())  # 段内按空行再切
        for p in paragraphs:
            if len(p) > 50:
                chunks.append(p.strip())
            elif chunks:  # 太短的段落合并到前一块
                chunks[-1] += "\n" + p.strip()
    return chunks
```

## 策略 4：语义切分

按语义边界（而非字符数）切分，保证每个分块语义完整。原理是计算相邻句子的 Embedding 相似度，相似度突变处即为语义边界：

```python
from langchain_experimental.text_splitter import SemanticChunker

semantic_splitter = SemanticChunker(
    embedding_function=get_embedding,
    breakpoint_threshold_type="percentile",  # 也可用 "standard_deviation" / "interquartile"
)

chunks = semantic_splitter.split_text(text)
```

注意：语义切分每次都要调用 Embedding API，速度较慢，适合离线处理。

## 策略 5：父子分块（高级）

兼顾检索精度与上下文完整性的高级策略。核心思路：**用小子块检索，返回对应的父块给 LLM**。

```
文档 → 父块（1000 tokens，保留上下文）→ 子块（200 tokens，用于向量化检索）
                      ↓
              命中子块 → 返回对应父块给 LLM
```

```python
class ParentChildChunker:
    def __init__(self, parent_size=1000, child_size=200, overlap=50):
        self.parent_splitter = RecursiveCharacterTextSplitter(
            chunk_size=parent_size, chunk_overlap=overlap)
        self.child_splitter = RecursiveCharacterTextSplitter(
            chunk_size=child_size, chunk_overlap=overlap // 2)

    def chunk(self, text: str) -> list:
        parents = self.parent_splitter.split_text(text)
        result = []
        for pi, parent in enumerate(parents):
            children = self.child_splitter.split_text(parent)
            for ci, child in enumerate(children):
                result.append({
                    "child_id": f"p{pi}c{ci}",
                    "child_text": child,       # 用于检索
                    "parent_id": f"p{pi}",
                    "parent_text": parent,     # 用于生成
                })
        return result
```

常见配置比例：**父块:子块 ≈ 5:1**（例如父块 1000 tokens，子块 200 tokens）。子块小所以检索精准，命中后返回父块保证上下文完整。

## 策略 6：基于稀有词率的边界检测（高级）

利用稀有词占比代理困惑度（Perplexity）来粗略检测语义边界——当一段文本中稀有词比例突然升高时，往往是话题转换点：

```python
import math, re

def rare_word_boundary_detection(text: str, max_len: int = 300) -> list:
    sentences = re.split(r'(?<=[。！？\n])', text)
    sentences = [s.strip() for s in sentences if s.strip()]
    chunks, start = [], 0
    for i in range(len(sentences)):
        window = "".join(sentences[start:i+1])
        rare_ratio = sum(1 for c in window if ord(c) > 0x4E00 and c not in "的了一是不了我在有") / max(len(window), 1)
        if len(window) > max_len or math.exp(rare_ratio * 5) > 50:
            chunks.append("".join(sentences[start:i+1]))
            start = i + 1
    if start < len(sentences):
        chunks.append("".join(sentences[start:]))
    return chunks
```

真实 PPL 需加载语言模型计算交叉熵损失，此处用稀有词占比模拟其变化趋势，展示边界检测思路。

## 切分策略对比

| 场景 | 推荐策略 | 块大小 | 重叠 |
|------|---------|-------|------|
| 技术文档（有标题） | 规则切分/语义切分 | 按段落 | 0 |
| 长文本小说 | 递归切分 | 500~1000 字 | 50~100 |
| 代码 | 按函数/类切分 | 单个函数 | 0 |
| 对话记录 | 按轮次切分 | 单轮对话 | 0 |
| 混合文档 | 递归切分 | 200~500 字 | 20~50 |
| **需要兼顾精度和上下文** | **父子分块** | **父1000/子200** | **50/25** |
| **追求语义完整性** | **稀有词边界检测** | **200~500** | **0** |

## 中文切分建议

中文没有空格分隔，切分时需按中文标点优先级排列：

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

chinese_splitter = RecursiveCharacterTextSplitter(
    chunk_size=300,          # 中文 300 字符 ≈ 英文 200 token
    chunk_overlap=50,
    separators=[             # 按中文习惯优先级排列
        "\n\n",              # 段落
        "\n",                # 行
        "。", "！", "？",    # 句子结束
        "；", "，", "、",     # 短句
        " ",                 # 词
        ""                   # 字符
    ],
    length_function=len      # 使用字符数而非 token 数
)
```

**经验值**：中文技术文档推荐 chunk_size=300、overlap=50，能在语义完整性和检索精度间取得较好平衡。
