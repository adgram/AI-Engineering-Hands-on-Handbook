## 一、查询重写

用户输入往往带有口语化表达、错别字或指代不清等问题，直接检索效果不佳。查询重写通过 LLM 将用户输入转化为更规范、语义更明确的检索查询，从而提升召回质量。

### 为什么要重写？

用户输入常含口语化、错别字或指代不清，直接检索容易漏掉相关文档。重写将输入转化为规范表达，使向量检索更准确地匹配知识库内容。

```
用户输入                 改写后
"RAG 是啥"      →    "RAG 检索增强生成是什么"
"那个 LLM 呢"   →    "大语言模型是什么"
"咋装"          →    "ChromaDB 的安装方法"
```

### 实现

以下实现了一个 QueryRewriter 类，通过 LLM 改写查询并支持生成多个同义查询变体以扩展召回。

```python
class QueryRewriter:
    def __init__(self, client):
        self.client = client
    
    def rewrite(self, query: str, context: str = None) -> str:
        """将查询改写为更规范的表达"""
        
        prompt = f"""你是一个查询优化助手。将用户的非正式查询改写为清晰、规范的搜索查询。

规则：
1. 补充术语全称（如 RAG→检索增强生成）
2. 消除口语化表达
3. 纠正错别字
4. 保持原意不变
5. 直接输出改写结果，不要解释

用户查询：{query}
改写后："""
        
        if context:
            prompt = f"对话上下文：{context}\n" + prompt
        
        response = self.client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=[{"role": "user", "content": prompt}],
        )
        
        return response.choices[0].message.content.strip()
    
    def expand_with_synonyms(self, query: str) -> list:
        """生成多个查询变体"""
        response = self.client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=[{"role": "user", "content": f"""为用户查询生成 3 个同义但表述不同的搜索查询，覆盖不同的用词和表达角度。

用户查询：{query}

每行一个查询，直接输出："""}],
        )
        
        variants = response.choices[0].message.content.strip().split("\n")
        return [query] + [v.strip() for v in variants if v.strip()]

# 使用
rewriter = QueryRewriter(client)
print(rewriter.rewrite("LLM 咋微调啊？"))
print(rewriter.rewrite("RAG 和 fine-tuning 哪个好？"))

# 多查询扩展
variants = rewriter.expand_with_synonyms("向量数据库的优点")
for v in variants:
    print(f"  - {v}")
```

### 多查询检索（Query Expansion）

对同一查询生成多个语义变体分别检索后合并结果，能有效提升查全率，特别适合表述单一但有多个相关方面的查询。

```python
def multi_query_search(collection, query: str, rewriter, k_per_query: int = 3) -> list:
    """用多个查询变体分别检索，合并结果"""
    queries = rewriter.expand_with_synonyms(query)
    
    all_results = {}
    for q in queries:
        results = collection.query(query_texts=[q], n_results=k_per_query)
        for doc_id, doc, dist in zip(results['ids'][0], results['documents'][0], results['distances'][0]):
            if doc_id not in all_results:
                all_results[doc_id] = {"doc": doc, "dist": dist, "count": 1}
            else:
                all_results[doc_id]["dist"] = min(all_results[doc_id]["dist"], dist)
                all_results[doc_id]["count"] += 1
    
    # 按距离排序
    sorted_results = sorted(all_results.items(), key=lambda x: x[1]["dist"])
    return sorted_results[:k_per_query]
```

## 二、查询分解

> 任务分解的 Prompt 技巧（Plan-and-Solve/ToT）详见 Prompt D2-P2，本篇是其在查询层面的特化应用

### 为什么要分解？

复合问题包含多个子问题，直接检索往往只能命中部分方面。分解为独立子问题分别检索，可确保每个维度都被覆盖。

```
复杂问题："RAG 和 Prompt Engineering 有什么区别？各自适合什么场景？"
→ 分解为：
  1. RAG 是什么？Prompt Engineering 是什么？
  2. RAG 和 Prompt Engineering 的区别
  3. RAG 适合什么场景？
  4. Prompt Engineering 适合什么场景？
```

### 实现

以下代码将复杂问题拆解为多个可直接检索的子问题，每步只检索一个简单方面。

```python
def decompose_query(query: str) -> list:
    """将复杂问题分解为多个子问题"""
    
    response = client.chat.completions.create(
        model="deepseek-v4-flash",
        messages=[{"role": "user", "content": f"""将以下复杂问题分解为 2-5 个简单子问题。每个子问题应该可以直接用于检索。

复杂问题：{query}

输出格式：每行一个子问题，不要编号以外的内容。

示例：
复杂问题：Python 和 JavaScript 的区别是什么？哪个更容易学？
子问题1: Python 编程语言的特点是什么？
子问题2: JavaScript 编程语言的特点是什么？
子问题3: Python 和 JavaScript 的主要区别
子问题4: Python 和 JavaScript 哪个更容易入门

复杂问题：{query}
子问题1:"""}],
    )
    
    text = response.choices[0].message.content
    sub_queries = []
    for line in text.strip().split("\n"):
        line = line.strip()
        if line and (line[0].isdigit() or line.startswith("子问题")):
            # 去掉 "子问题1:" 或 "1." 前缀
            if ":" in line:
                line = line.split(":", 1)[1].strip()
            elif ". " in line:
                line = line.split(". ", 1)[1].strip()
            sub_queries.append(line)
    
    return sub_queries

# 使用
queries = decompose_query("RAG 和 Prompt Engineering 有什么区别？各自适合什么场景？")
for i, q in enumerate(queries, 1):
    print(f"子问题{i}: {q}")
```

### 子问题检索 + 答案合并

将子问题的检索结果合并后，由 LLM 综合所有信息生成完整回答，确保复合问题的每个部分都被覆盖。

```python
class DecompositionRAG:
    def __init__(self, collection, client):
        self.collection = collection
        self.client = client
    
    def ask(self, query: str) -> str:
        # 分解
        sub_queries = decompose_query(query)
        print(f"分解为 {len(sub_queries)} 个子问题")
        
        # 对每个子问题检索
        all_contexts = []
        for sq in sub_queries:
            results = self.collection.query(query_texts=[sq], n_results=2)
            for doc in results['documents'][0]:
                if doc not in all_contexts:
                    all_contexts.append(doc)
        
        print(f"共检索到 {len(all_contexts)} 个相关文档块")
        
        # 合并生成
        context = "\n\n".join([f"[{i+1}] {c}" for i, c in enumerate(all_contexts)])
        
        response = self.client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=[{"role": "user", "content": f"""请基于以下参考资料，全面回答用户的问题。

参考资料：
{context}

问题：{query}

请确保覆盖问题的所有方面。"""}],
        )
        
        return response.choices[0].message.content
```

## 两种技术对比

重写适合标准化输入，扩展适合提升查全率，分解适合多跳复合问题，应根据具体需求选择使用。

| 技术 | 适用场景 | 效果 | 成本 |
|------|---------|------|------|
| 查询重写 | 非标准表达、口语化 | 检索质量提升 10-30% | 低 |
| 多查询扩展 | 查全率优先 | 查全率提升，查准率可能下降 | 中 |
| 查询分解 | 复杂多跳问题 | 对复合问题效果好 | 高 |

## 三、A-RAG 三层检索工具设计（归属 P9 A-RAG 专题）

> 参考：Context Tracker 的基础实现在 Day4 P2，本节是其在 A-RAG 场景的扩展应用

来自 Agentic RAG（A-RAG）的设计思路：给 LLM 提供三个不同粒度的检索工具，让模型自主决定如何检索：

```python
class ARAGRetrievalTools:
    """A-RAG 三层检索工具：关键词 + 语义 + 文档块读取"""
    
    def __init__(self, collection, llm_client):
        self.collection = collection
        self.client = llm_client
        self.context_tracker = set()  # 已读 ID
    
    def keyword_search(self, query: str, top_k: int = 5) -> list:
        """工具1：关键词搜索（精确匹配）"""
        results = self.collection.query(query_texts=[query], n_results=top_k)
        snippets = []
        for i, (doc, dist) in enumerate(zip(results['documents'][0], results['distances'][0])):
            snippets.append({
                "id": results['ids'][0][i],
                "snippet": doc[:150],  # 只返回预览片段
                "type": "keyword",
                "score": 1 - dist,
            })
        return snippets
    
    def semantic_search(self, query: str, top_k: int = 5) -> list:
        """工具2：语义搜索（向量匹配）"""
        return self.keyword_search(query, top_k)  # 简化为同接口
    
    def read_chunk(self, chunk_id: str) -> str:
        """工具3：读取完整文档块"""
        if chunk_id in self.context_tracker:
            return f"[已读] 文档块 {chunk_id} 之前已读取过，不再重复"
        result = self.collection.get(ids=[chunk_id])
        self.context_tracker.add(chunk_id)
        if result['documents']:
            return result['documents'][0]
        return "未找到该文档块"
    
    def retrieve(self, query: str) -> list:
        """主入口：先用关键词+语义搜索获取snippet，再读取完整内容"""
        kw_results = self.keyword_search(query, top_k=3)
        sem_results = self.semantic_search(query, top_k=3)
        
        # 合并去重
        seen = set()
        full_docs = []
        for r in kw_results + sem_results:
            rid = r["id"]
            if rid not in seen:
                seen.add(rid)
                full_content = self.read_chunk(rid)
                if not full_content.startswith("[已读]"):
                    full_docs.append({"id": rid, "content": full_content})
        
        return full_docs

# 使用：先通过 snippet 预览判断相关性，再决定是否读取完整文档
tools = ARAGRetrievalTools(collection, client)
docs = tools.retrieve("RAG 的工作原理是什么？")
for d in docs:
    print(f"[{d['id']}] {d['content'][:80]}...")
```

**设计核心**：snippet（预览） + 完整读取分离，模型先看 snippet 判断是否相关，再决定是否深入阅读。

## 四、ReAct 循环与预算控制（ 归属 P9 A-RAG 专题）

在检索循环中加入 Token 预算和最大迭代限制，防止无限循环：

```python
# 注：此处用 DeepSeek 模型，tokenizer 估算仅供参考
# import tiktoken; enc = tiktoken.encoding_for_model("gpt-4o")

class ReActBudgetController:
    """ReAct 循环的预算控制"""
    
    def __init__(self, max_loops=10, max_token_budget=128000):
        self.max_loops = max_loops
        self.max_token_budget = max_token_budget
        self.loop_count = 0
        self.token_usage = 0
        self.tokenizer = None  # 接入 DeepSeek tokenizer 或使用 tiktoken 近似估算
    
    def can_continue(self, messages: list) -> bool:
        self.loop_count += 1
        
        # 检查最大循环
        if self.loop_count > self.max_loops:
            print(f"达到最大循环 {self.max_loops}，强制结束")
            return False
        
        # 检查 Token 预算
        total_tokens = sum(len(self.tokenizer.encode(str(m.get("content", "")))) for m in messages)
        if total_tokens > self.max_token_budget:
            print(f"Token 预算超限 ({total_tokens}>{self.max_token_budget})，强制结束")
            return False
        
        return True
    
    def force_final_answer(self, messages: list, llm_client) -> str:
        """预算超限时强制生成最终回答"""
        force_prompt = "你已达到检索限制。请基于已有信息给出最终答案，不要再调用任何工具。"
        messages.append({"role": "user", "content": force_prompt})
        response = llm_client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=messages,
            temperature=0.0
        )
        return response.choices[0].message.content

# 使用
budget = ReActBudgetController(max_loops=5, max_token_budget=64000)
messages = [{"role": "user", "content": "检索 RAG 相关信息"}]

while budget.can_continue(messages):
    # ... 检索 + 生成循环 ...
    # 如果模型决定返回答案，break
    # 如果达到限制，budget.force_final_answer(messages, client)
    break  # 简化示例
```
