# P3：文档切分（Chunking）策略

## 目标
掌握不同文档切分方法及其对检索效果的影响

## 为什么需要切分？

- LLM 上下文窗口有限（不能把所有文档塞进去）
- 检索需要细粒度的片段才能精确匹配
- 切分粒度直接影响检索质量

## 切分策略对比

### 1. 固定长度切分

```python
def fixed_length_chunk(text: str, chunk_size: int = 200, overlap: int = 20) -> list:
    """固定字符数切分，带重叠"""
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        if chunk:
            chunks.append(chunk.strip())
        start += (chunk_size - overlap)
    return chunks

text = "很长很长的文档内容..." * 100
chunks = fixed_length_chunk(text, chunk_size=200, overlap=20)
print(f"切分出 {len(chunks)} 个块")
print(f"第一块: {chunks[0][:50]}...")
print(f"第二块: {chunks[1][:50]}...")
```

**优点**：简单、可预测
**缺点**：可能切断句子/段落语义

### 2. 递归字符切分（推荐）

LangChain 提供的 `RecursiveCharacterTextSplitter`，按优先级分隔符逐级切分：

```bash
pip install langchain-text-splitters
```

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text = """
## 引言
RAG（Retrieval-Augmented Generation）是一种结合检索和生成的技术。

## 工作原理
RAG 的工作流程分为三步：
1. 检索：从知识库中找到相关文档
2. 增强：将检索到的文档和问题一起组成 Prompt
3. 生成：LLM 根据增强后的 Prompt 生成回答

## 优势
相比传统 LLM，RAG 有以下优势：
- 知识实时更新
- 可追溯信息来源
- 降低幻觉风险
"""

splitter = RecursiveCharacterTextSplitter(
    chunk_size=100,
    chunk_overlap=20,
    separators=["\n\n", "\n", "。", "，", " ", ""]  # 由大到小
)

chunks = splitter.split_text(text)
for i, chunk in enumerate(chunks):
    print(f"块 {i+1} ({len(chunk)}字): {chunk}")
```

### 3. 基于规则的段落切分

按标题和段落边界切分，不跨语义单元：

```python
import re

def rule_based_chunk(text: str) -> list:
    """按标题和段落切分"""
    # 按 ## 标题切分
    sections = re.split(r'(?=##\s)', text)
    chunks = []
    
    for section in sections:
        if not section.strip():
            continue
        # 每个 section 内部按空行再切
        paragraphs = re.split(r'\n\s*\n', section.strip())
        for p in paragraphs:
            if len(p) > 50:  # 太短的段落合并到前一块
                chunks.append(p.strip())
            elif chunks:
                chunks[-1] += "\n" + p.strip()
    
    return chunks

chunks = rule_based_chunk(text)
for i, chunk in enumerate(chunks):
    print(f"语义块 {i+1} ({len(chunk)}字): {chunk[:60]}...")
```

### 4. 语义切分（LangChain SemanticChunker）

按语义边界（而非字符数）切分，保证每个分块语义完整：

```bash
pip install langchain-experimental
```

```python
from langchain_experimental.text_splitter import SemanticChunker
from openai import OpenAI
import os

# 需要 Embedding 模型来计算语义相似度
embeddings = OpenAI(
    api_key=os.getenv("SILICONFLOW_API_KEY"),
    base_url="https://api.siliconflow.cn/v1"
)

def get_embedding(text):
    resp = embeddings.embeddings.create(model="BAAI/bge-m3", input=text)
    return resp.data[0].embedding

# 语义切分器
semantic_splitter = SemanticChunker(
    embedding_function=get_embedding,
    breakpoint_threshold_type="percentile",  # 也可用 "standard_deviation" / "interquartile"
)

text = """
## 引言
RAG（Retrieval-Augmented Generation）是一种结合检索和生成的技术。
它通过先检索知识库再生成回答的方式，有效减少了 LLM 的幻觉问题。

## 工作原理
RAG 的工作流程分为三步：
1. 检索：从知识库中找到相关文档
2. 增强：将检索到的文档和问题一起组成 Prompt
3. 生成：LLM 根据增强后的 Prompt 生成回答
"""

chunks = semantic_splitter.split_text(text)
for i, chunk in enumerate(chunks):
    print(f"语义块 {i+1} ({len(chunk)}字): {chunk[:80]}...")
```

**注意**：语义切分每次都要调用 Embedding API，速度较慢，适合离线处理。

### 5. 父子分块（Parent-Child Chunking） 高级策略

兼顾检索精度与上下文完整性的高级策略。核心思路：用小子块检索，返回对应的父块给 LLM。

```
文档 → 父块（1000 tokens，保留上下文）→ 子块（200 tokens，用于向量化检索）
                      ↓
              命中子块 → 返回对应父块给 LLM
```

```python
class ParentChildChunker:
    """父子分块：子块用于检索，父块用于生成"""
    
    def __init__(self, parent_size=1000, child_size=200, overlap=50):
        from langchain_text_splitters import RecursiveCharacterTextSplitter
        self.parent_splitter = RecursiveCharacterTextSplitter(
            chunk_size=parent_size, chunk_overlap=overlap
        )
        self.child_splitter = RecursiveCharacterTextSplitter(
            chunk_size=child_size, chunk_overlap=overlap // 2
        )
    
    def chunk(self, text: str) -> list:
        """返回父子块对"""
        parents = self.parent_splitter.split_text(text)
        result = []
        for pi, parent in enumerate(parents):
            children = self.child_splitter.split_text(parent)
            for ci, child in enumerate(children):
                result.append({
                    "child_id": f"p{pi}c{ci}",
                    "child_text": child,
                    "parent_id": f"p{pi}",
                    "parent_text": parent,
                })
        return result

# 使用：检索时匹配子块，生成时喂父块
chunker = ParentChildChunker(parent_size=1000, child_size=200)
pairs = chunker.chunk("很长很长的文档..." * 50)
print(f"父子块对数: {len(pairs)}")
print(f"子块平均长度: {sum(len(p['child_text']) for p in pairs) / len(pairs):.0f} 字")
```

常见配置比例：**父块:子块 ≈ 5:1**（例如父块 1000 tokens，子块 200 tokens）。

### 6. 基于稀有词率的边界检测（模拟 PPL）  高级策略

利用稀有词占比代理困惑度（Perplexity）来粗略检测语义边界：

```python
# 注：真实 PPL 需加载语言模型（如 GPT-2）计算交叉熵损失
# 此处仅用稀有词占比模拟 PPL 的变化趋势，展示边界检测思路
import math, re

def rare_word_boundary_detection(text: str, max_len: int = 300) -> list:
    sentences = re.split(r'(?<=[。！？\n])', text)
    sentences = [s.strip() for s in sentences if s.strip()]
    chunks, start = [], 0
    for i in range(len(sentences)):
        window = "".join(sentences[start:i+1])
        rare_ratio = sum(1 for c in window if ord(c) > 0x4E00 and c not in "的了一是不了我在有") / max(len(window), 1)
        if len(window) > max_len or math.exp(rare_ratio * 5) > 50:
            chunks.append("".join(sentences[start:i+1]))
            start = i + 1
    if start < len(sentences):
        chunks.append("".join(sentences[start:]))
    return chunks

text = "RAG 是检索增强生成。它结合了检索和生成。Prompt Engineering 是提示工程。向量数据库用于存储向量。"
for i, c in enumerate(rare_word_boundary_detection(text, max_len=50)):
    print(f"块 {i+1}: {c}")
```

## 切分参数实验

```python
def evaluate_chunking_strategy(text: str, chunk_func, **kwargs):
    """评估切分策略"""
    chunks = chunk_func(text, **kwargs)
    
    print(f"切分策略: {chunk_func.__name__}")
    print(f"参数: {kwargs}")
    print(f"切分数量: {len(chunks)}")
    print(f"平均长度: {sum(len(c) for c in chunks) / len(chunks):.0f} 字")
    print(f"最短: {min(len(c) for c in chunks)} 字")
    print(f"最长: {max(len(c) for c in chunks)} 字")
    
    # 检查语义完整性
    incomplete = sum(1 for c in chunks if not c.endswith(('。', '！', '？', '\n')))
    print(f"截断片段数: {incomplete}/{len(chunks)}")
    print()

# 对比不同策略
long_text = "RAG 技术。\n\n" * 50 + "Prompt Engineering 技术。\n\n" * 50

evaluate_chunking_strategy(long_text, fixed_length_chunk, chunk_size=50, overlap=10)
evaluate_chunking_strategy(long_text, rule_based_chunk)
```

## 最佳实践

| 场景 | 推荐策略 | 块大小 | 重叠 |
|------|---------|-------|------|
| 技术文档（有标题） | 语义切分 | 按段落 | 0 |
| 长文本小说 | 递归切分 | 500~1000 字 | 50~100 |
| 代码 | 按函数/类切分 | 单个函数 | 0 |
| 对话记录 | 按轮次切分 | 单轮对话 | 0 |
| 混合文档 | 递归切分 | 200~500 字 | 20~50 |
| **需要兼顾精度和上下文** | **父子分块** | **父1000/子200** | **50/25** |
| **追求语义完整性** | **稀有词边界检测（模拟PPL）** | **200~500** | **0** |
