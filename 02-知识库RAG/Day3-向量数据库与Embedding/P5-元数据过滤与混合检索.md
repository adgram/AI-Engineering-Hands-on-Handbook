## 元数据过滤

### ChromaDB 支持的条件过滤

```python

# IN 操作

# AND 条件
where={"$and": [{"topic": "RAG"}, {"year": {"$gte": 2023}}]}

# OR 条件
where={"$or": [{"topic": "RAG"}, {"topic": "Agent"}]}
```

## 混合检索（Hybrid Search）

向量搜索擅长语义匹配，但关键词搜索在精确匹配上更好。混合检索结合两者优势。

### 方案 1：ChromaDB 内置关键词（基础）

目前 ChromaDB 主要靠向量搜索，但可以通过 `where` 条件做关键词匹配：

```python
# 用 metadata 做精确关键词匹配
where={"topic": "RAG"}  # 精确匹配 topic 字段
```

### 方案 2：结合外部全文搜索引擎

```python
import sqlite3
import json

class HybridRetriever:
    """向量 + 关键词混合检索"""
    
    def __init__(self, collection):
        self.collection = collection
        # SQLite 存储关键词索引
        self.conn = sqlite3.connect(":memory:")
        self.conn.execute("CREATE TABLE keyword_index (id TEXT, content TEXT)")
    
    # add_documents 省略：collection.add 逻辑与 P2 相同
    # 实际使用时调用 self.collection.add() 添加向量数据
    # 再通过 self.conn.execute("INSERT INTO ...") 添加关键词索引

    def hybrid_search(self, query: str, n_results: int = 5, keyword_weight: float = 0.3):
        # 向量搜索
        vector_results = self.collection.query(
            query_texts=[query],
            n_results=n_results * 2  # 多取一些用于融合
        )
        
        # 关键词搜索（简单包含匹配）
        keyword_results = self.conn.execute(
            "SELECT id, content FROM keyword_index WHERE content LIKE ?",
            (f"%{query}%",)
        ).fetchall()
        
        # 融合评分（简化版：取并集）
        combined = {}
        
        # 向量评分
        for i, doc_id in enumerate(vector_results['ids'][0]):
            score = (1 - keyword_weight) * (1 - i / len(vector_results['ids'][0]))
            combined[doc_id] = score
        
        # 关键词评分
        kw_count = len(keyword_results)
        for j, (doc_id, _) in enumerate(keyword_results):
            score = keyword_weight * (1 - j / max(kw_count, 1))
            combined[doc_id] = combined.get(doc_id, 0) + score
        
        # 按总分排序
        sorted_results = sorted(combined.items(), key=lambda x: x[1], reverse=True)
        return sorted_results[:n_results]
```

### 方案 3：RRF（Reciprocal Rank Fusion）融合

RRF 是业界常用的混合检索融合算法：

```python
def rrf_fusion(vector_results, keyword_results, k=60):
    """
    RRF 融合算法
    k 是常数（通常 60），防止极端排序影响
    """
    scores = {}
    
    for rank, doc_id in enumerate(vector_results, 1):
        scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + rank)
    
    for rank, doc_id in enumerate(keyword_results, 1):
        scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + rank)
    
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)

# 模拟使用
vector_rankings = ["doc_1", "doc_2", "doc_3", "doc_4", "doc_5"]
keyword_rankings = ["doc_3", "doc_1", "doc_5", "doc_7", "doc_9"]

final_ranking = rrf_fusion(vector_rankings, keyword_rankings)
print("RRF 融合后排序:")
for doc_id, score in final_ranking:
    print(f"  {doc_id}: {score:.4f}")
```

## 文档去重 — 四层去重体系

重复文档会导致检索噪声，降低回答质量。企业级 RAG 需要多层去重：

```
第一层：文件级    → SHA256 文件哈希，相同文件只处理一次
第二层：内容级    → 忽略格式差异，归一化内容后计算 MD5（跨格式去重）
第三层：Segment级 → 每个文本块独立哈希，避免重复块入库
第四层：结果折叠  → 检索后对同一来源的结果折叠，只保留最佳匹配
```

以下实现第一层（文件级）和第三层（Segment 级），二、四层原理类似不再重复：

```python
import hashlib

class DedupEngine:
    """去重引擎——文件级 + Segment 级"""
    
    def __init__(self):
        self.file_hashes = set()      # 已处理的文件指纹
        self.segment_hashes = set()   # 已入库的块指纹
    
    def is_duplicate_file(self, filepath: str) -> bool:
        with open(filepath, "rb") as f:
            fhash = hashlib.sha256(f.read()).hexdigest()
        if fhash in self.file_hashes:
            return True
        self.file_hashes.add(fhash)
        return False
    
    def is_duplicate_segment(self, segment: str) -> bool:
        shash = hashlib.md5(segment.encode()).hexdigest()
        if shash in self.segment_hashes:
            return True
        self.segment_hashes.add(shash)
        return False

dedup = DedupEngine()
chunks = chunk_document(content)
for chunk in chunks:
    if not dedup.is_duplicate_segment(chunk):
        collection.add(documents=[chunk], ids=[...])
```

## Modular RAG 视角（完整讲解见 Day5）

元数据过滤和混合检索是 Modular RAG（模块化 RAG）的典型组件。Modular RAG 将 RAG 系统拆分为可插拔模块：

```
┌─ 查询处理模块 ─┐    ┌─ 检索模块 ────┐    ┌─ 增强模块 ────┐    ┌─ 生成模块 ────┐
│ - 查询重写    │  → │ - 向量检索     │ → │ - 重排序      │ →  │ - LLM 生成    │
│ - 查询扩展    │    │ - 关键词检索   │   │ - 上下文压缩   │    │ - 结构化输出   │
│ - HyDE       │    │ - 元数据过滤   │   │ - 结果融合    │    │ - 引用标注    │
└──────────────┘    └──────────────┘    └─────────────┘    └──────────────┘
```

每个模块可独立替换、A/B 测试，这就是你在本课中实践的元数据过滤（检索模块）和 RRF 融合（增强模块）的工程意义。

## 工业实践：检索结果质量过滤

字节跳动的三层过滤体系，用可调参数来控制结果质量：

```python
class QualityFilter:
    """检索结果质量过滤：静态规则 + 动态调整"""
    
    def __init__(self, min_score: float = 0.7, max_results: int = 5):
        self.min_score = min_score          # 相似度最低阈值
        self.authority_boost = {            # 来源权威性排序
            "official": 1.0,                # 官方文档，原样保留
            "partner": 0.9,                 # 合作机构，轻微降权
            "user": 0.7,                    # 用户生成内容，降权
        }
    
    def filter(self, results: list, source_type: str = "official") -> list:
        """多层过滤：分数 + 来源 + 去重"""
        # 第一层：相似度过滤
        filtered = [r for r in results if r["score"] >= self.min_score]
        
        # 第二层：来源加权排序
        boost = self.authority_boost.get(source_type, 0.8)
        for r in filtered:
            r["adjusted_score"] = r["score"] * boost
        
        # 第三层：冗余合并（同一文档的相邻片段合并）
        merged = []
        seen_docs = set()
        for r in sorted(filtered, key=lambda x: x["adjusted_score"], reverse=True):
            doc_id = r.get("doc_id", "")
            if doc_id not in seen_docs:
                seen_docs.add(doc_id)
                merged.append(r)
        
        return merged[:self.max_results]

# 使用：官方文档取≥0.7分的结果，用户内容取≥0.8分
official_filter = QualityFilter(min_score=0.7)
user_filter = QualityFilter(min_score=0.8)

# 实际效果参考（字节跳动生产数据）：
# - 静态过滤：来源权威性 + 时效性 + 质量评分
# - 动态过滤：用户标记无效超10次/天，自动提高该文档阈值+1
```
