## 评估维度

生成质量可以从多个角度衡量，以下四个核心维度覆盖了回答的准确性和用户体验。

| 维度 | 定义 | 评估方式 |
|------|------|----------|
| **Faithfulness** | 回答是否忠实于检索到的资料 | 检查回答是否有资料不支持的陈述 |
| **Relevance** | 回答是否回答了用户的问题 | 检查回答和问题的相关性 |
| **Completeness** | 回答是否完整覆盖问题需求 | 检查是否有遗漏 |
| **Conciseness** | 回答是否简洁 | 检查冗余度 |

## 方法一：LLM-as-Judge（用 LLM 评估 LLM）

利用 LLM 自身对 Faithfulness 和 Relevance 进行打分，是一种自动化且可扩展的评估策略，无需人工逐条标注。

```python
import json

class RAGEvaluator:
    def __init__(self, llm, model: str = "deepseek-v4-flash"):
        self.llm = llm
        self.model = model

    def check_faithfulness(self, answer: str, context: str) -> dict:
        prompt = f"""你是一个评估助手。判断以下回答是否完全基于给定的上下文，没有编造或幻觉。

上下文：
{context}

回答：
{answer}

请输出 JSON：
{{
    "faithfulness_score": 0-10,
    "is_hallucination": true/false,
    "hallucinated_parts": ["编造的内容1", "编造的内容2"],
    "reason": "简要说明"
}}"""

        resp = self.llm.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            response_format={"type": "json_object"},  # 结构化输出方法详见 Prompt D1-P5
        )
        return json.loads(resp.choices[0].message.content)

    def check_relevance(self, answer: str, question: str) -> dict:
        prompt = f"""判断以下回答是否有效回答了用户的问题。

问题：{question}
回答：{answer}

评分标准：
- 10: 完全正确回答了问题，信息充分
- 7-9: 回答了问题，但可以更完善
- 4-6: 部分相关，但没有直接回答
- 1-3: 几乎不相关
- 0: 完全不相关

输出 JSON：{{"relevance_score": 0-10, "reason": "..."}}"""

        resp = self.llm.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            response_format={"type": "json_object"},
        )
        return json.loads(resp.choices[0].message.content)

    def evaluate(self, question: str, answer: str, context: str) -> dict:
        faithfulness = self.check_faithfulness(answer, context)
        relevance = self.check_relevance(answer, question)

        return {
            "question": question,
            "answer": answer,
            "faithfulness": faithfulness,
            "relevance": relevance,
            "overall": (faithfulness["faithfulness_score"] + relevance["relevance_score"]) / 2
        }
```

## 端到端 RAG Pipeline 评估

将检索评估与生成评估串联起来，对完整的 RAG 流程进行端到端测试，输出统一的平均分和幻觉率报告。

```python
import numpy as np

def evaluate_rag_pipeline(rag, evaluator, test_cases: list) -> dict:
    all_results = []

    for case in test_cases:
        result = rag.ask(case["question"])
        eval_result = evaluator.evaluate(
            case["question"],
            result["answer"],
            "\n".join(result["sources"])
        )

        all_results.append({
            **case,
            "rag_answer": result["answer"],
            "sources": result["sources"],
            "evaluation": eval_result
        })

    avg_faithfulness = np.mean([r["evaluation"]["faithfulness"]["faithfulness_score"] for r in all_results])
    avg_relevance = np.mean([r["evaluation"]["relevance"]["relevance_score"] for r in all_results])

    return {
        "avg_faithfulness": avg_faithfulness,
        "avg_relevance": avg_relevance,
        "avg_overall": (avg_faithfulness + avg_relevance) / 2,
        "hallucination_rate": sum(1 for r in all_results if r["evaluation"]["faithfulness"]["is_hallucination"]) / len(all_results),
        "details": all_results
    }
```

## 常见评估陷阱

使用 LLM-as-Judge 进行自动化评估时，需要注意以下几个常见的系统性偏差和问题。

1. **评估偏差**：LLM-as-Judge 倾向于给更长的回答更高分
2. **答非所问**：回答很精确但没有回答问题（低分）
3. **过度自信**：回答错了但很流畅（Low faithfulness）
4. **同义表达**：LLM 无法识别同义表达的答案

## 工业实践：全链路质量控制

字节跳动的生成质量管控体系分为"事前-事中-事后"三个阶段：

```python
class QualityControlPipeline:
    """全链路质量控制：事前预防→事中监控→事后优化"""

    def __init__(self, llm, model: str = "deepseek-v4-flash"):
        self.llm = llm
        self.model = model

    def pre_generation(self, query: str, context: str) -> str:
        prompt = f"""基于以下资料回答问题。每句话必须对应资料中的具体内容。
资料：{context[:800]}
问题：{query}
回答（请标注每句话对应的资料片段编号）："""

        return prompt

    def during_generation(self, response: str, context: str) -> bool:
        context_sentences = set(s.strip() for s in context.split("。") if len(s) > 5)
        response_sentences = set(s.strip() for s in response.split("。") if len(s) > 5)

        overlap = len(context_sentences & response_sentences)
        if overlap < 2:
            return False
        return True

    def post_generation(self, response: str, context: str) -> dict:
        resp = self.llm.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": f"""校验以下回答是否有信息不在资料中。

资料：{context[:600]}

回答：{response}

输出 JSON：
- "hallucination": 是否存在幻觉（true/false）
- "error_sentences": 有问题的句子列表
- "correction": 修正建议"""}],
            response_format={"type": "json_object"},
        )
        return json.loads(resp.choices[0].message.content)
```
