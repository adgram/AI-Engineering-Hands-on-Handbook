## 成本计算公式

RAG 系统的费用由输入和输出两部分 Token 用量乘以对应单价构成，以下公式是成本估算的基础。

```
总费用 = Prompt Tokens × 输入价格 + Completion Tokens × 输出价格

DeepSeek Chat（参考）：
  输入：¥0.14 / 1M tokens
  输出：¥0.28 / 1M tokens
```

## 1. Token 用量监控

TokenMonitor 类可以记录每次 API 调用的 Token 消耗和延迟，并自动生成费用估算报告，让 RAG 成本透明可追踪。

```python
import os, time
from collections import defaultdict
from datetime import datetime
import json

class TokenMonitor:
    def __init__(self, log_dir="./logs"):
        self.log_dir = log_dir
        self.sessions = defaultdict(lambda: {
            "prompt_tokens": 0, "completion_tokens": 0,
            "calls": 0, "total_time": 0
        })
        os.makedirs(log_dir, exist_ok=True)
    
    def log_call(self, session: str, prompt_tokens: int, completion_tokens: int, elapsed: float):
        s = self.sessions[session]
        s["prompt_tokens"] += prompt_tokens
        s["completion_tokens"] += completion_tokens
        s["calls"] += 1
        s["total_time"] += elapsed
    
    def cost_estimate(self, session: str = None) -> dict:
        """估算费用"""
        input_price = 0.14 / 1_000_000  # ¥/token
        output_price = 0.28 / 1_000_000
        
        if session:
            sessions = {session: self.sessions[session]}
        else:
            sessions = dict(self.sessions)
        
        report = {}
        for name, data in sessions.items():
            cost = data["prompt_tokens"] * input_price + data["completion_tokens"] * output_price
            report[name] = {
                **data,
                "estimated_cost_yuan": round(cost, 4),
            }
        
        return report
    
    def summary(self):
        total = self.cost_estimate()
        all_data = self.sessions
        t_prompt = sum(d["prompt_tokens"] for d in all_data.values())
        t_completion = sum(d["completion_tokens"] for d in all_data.values())
        t_calls = sum(d["calls"] for d in all_data.values())
        
        return {
            "total_calls": t_calls,
            "total_prompt_tokens": t_prompt,
            "total_completion_tokens": t_completion,
            "total_tokens": t_prompt + t_completion,
            "total_cost": total
        }

# 集成到 LLM 客户端（基类为 OpenAI，来自 openai 库）
from openai import OpenAI

class CostAwareLLMClient(OpenAI):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.monitor = TokenMonitor()
        self.current_session = "default"
    
    def chat(self, messages, **kwargs):
        start = time.time()
        response = super().chat.completions.create(model="deepseek-v4-flash", messages=messages, **kwargs)
        elapsed = time.time() - start
        
        self.monitor.log_call(
            self.current_session,
            response.usage.prompt_tokens,
            response.usage.completion_tokens,
            elapsed
        )
        
        return response
    
    def print_cost_report(self):
        report = self.monitor.summary()
        print("=" * 40)
        print("Token 用量报告")
        print("=" * 40)
        print(f"总调用次数: {report['total_calls']}")
        print(f"总输入 Tokens: {report['total_prompt_tokens']:,}")
        print(f"总输出 Tokens: {report['total_completion_tokens']:,}")
        print(f"总计 Tokens: {report['total_tokens']:,}")
        print(f"估算费用: ¥{report['total_cost']:.2f}")
```

## 2. 成本优化策略

在建立了成本监控之后，下一步就是主动优化。优化可以从四个维度入手：精简 Prompt 减少输入 Token、压缩历史减少累积消耗、选择合适模型平衡性价比、控制输出长度避免浪费。

### 策略 A：精简 Prompt

Prompt 中多余的修饰词和指令会增加不必要的输入 Token 消耗，精简后的 Prompt 可以显著降低成本。

```python
# 冗余
prompt = f"你是一个非常有经验的并且知识渊博的AI助手，请帮助用户回答以下问题。用户的问题是：'{query}'请给出详细回答。"

# 精简
prompt = f"回答：{query}"
```

### 策略 B：压缩历史

多轮对话中累积的历史内容会持续消耗 Token，通过 LLM 将历史压缩为摘要可以避免 Token 浪费。

```python
# ~ 代替全文历史
def compress_history(history: list, max_tokens: int = 200) -> str:
    """用 LLM 压缩对话历史"""
    if not history:
        return ""
    
    text = json.dumps(history, ensure_ascii=False)
    response = client.chat.completions.create(
        model="deepseek-v4-flash",
        messages=[{"role": "user", "content": f"将以下对话压缩到{max_tokens}tokens以内，保留关键信息：{text}"}],
        max_tokens=max_tokens
    )
    return response.choices[0].message.content
```

> 历史压缩的完整原理（GSSC-Compress 步骤）见 Prompt D2-P1

### 策略 C：选择合适的模型

简单任务使用更便宜的小模型，复杂任务调用更强的模型，这种分级策略可以在保证质量的同时降低成本。

```python
# 简单任务用小模型，复杂任务用大模型
def smart_model(question: str) -> str:
    if len(question) < 20:  # 简单问题
        return "deepseek-chat"  # 便宜的
    return "deepseek-v4-flash"  # 更强的
```

### 策略 D：限制输出长度

通过设置 max_tokens 参数限制模型输出长度，避免生成冗长内容造成 Token 浪费。

```python
# 控制 max_tokens
response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[{"role": "user", "content": query}],
    max_tokens=500,  # 限制输出长度
)
```

## 3. Token 利用率分析

通过分析输出/输入 Token 比和单次调用平均 Token 消耗，可以评估 Token 使用效率并发现优化空间。

```python
def analyze_token_usage(session_data: dict) -> dict:
    """分析 Token 使用效率"""
    total = session_data["prompt_tokens"] + session_data["completion_tokens"]
    
    # 输出/输入比
    ratio = session_data["completion_tokens"] / max(session_data["prompt_tokens"], 1)
    
    # 每次调用的平均 Token
    avg_per_call = total / max(session_data["calls"], 1)
    
    return {
        "output_input_ratio": round(ratio, 2),
        "avg_tokens_per_call": int(avg_per_call),
        "total_tokens": total,
    }
```

## 工业实践：字节跳动成本精细化管控

字节跳动在生产环境中对 RAG 系统进行了系统化的成本管控，以下从成本构成、优化策略、模型量化和资源调度四个层面拆解其做法。

### 成本构成拆解

字节跳动的实践表明，RAG 系统的成本主要由 GPU 推理、向量存储和数据传输三部分构成，下表展示了各成本项的占比。

| 成本项 | 占比 | 说明 |
|-------|------|------|
| 计算（GPU推理） | 60% | Embedding + 生成模型推理 |
| 存储（向量库） | 25% | 向量存储 + 索引文件 |
| 网络（数据传输） | 15% | 跨区域同步 + 用户请求 |

### 生产级优化策略

在成本构成明确之后，字节跳动从模型量化和资源调度两方面入手进行生产级优化，具体方案如下。

### 模型量化方案（字节跳动 ByteInfer 框架）

量化是降低推理成本的有效手段，字节跳动 ByteInfer 框架支持 FP16 和 INT8 两种精度，在下表中对比其收益与质量损失。

| 精度 | 显存节省 | 速度提升 | 质量损失 |
|------|---------|---------|---------|
| FP16 | 50% | 1.4x | <3% |
| INT8（QAT） | 75% | 2x | <5% |

### 三级资源调度（字节跳动生产标准）

字节跳动根据业务优先级将 GPU 资源划分为三级，不同场景分配不同的计算资源以优化总成本。

| 优先级 | 场景 | 资源策略 |
|-------|------|---------|
| 高优 | 抖音实时客服 | 专属 GPU 集群，latency < 1s |
| 中优 | 飞书知识库 | 共享集群，latency < 3s |
| 低优 | 批量总结 | 夜间空闲资源，成本 ↓ 70% |

### 成本估算公式

基于日均查询量和平均 Token 消耗，可以用以下公式快速估算 RAG 系统的月度运行成本。

```python
def estimate_monthly_cost(daily_queries: int, avg_tokens_per_query: int, 
                          model_cost_per_1k_tokens: float = 0.002):
    monthly_tokens = daily_queries * avg_tokens_per_query * 30
    cost_usd = monthly_tokens / 1000 * model_cost_per_1k_tokens
    return f"预估月成本：${cost_usd:.2f}（{daily_queries}查询/天）"

print(estimate_monthly_cost(10000, 2000))
```
